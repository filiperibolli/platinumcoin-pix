# Step 43 — HTML API explorer (unified swagger-like page)

## Objective
`tools/api-explorer/index.html`: a **single self-contained HTML file** (vanilla JS + embedded CSS, no build step, no server — open from disk) presenting every API of the platform in one swagger-style page, grouped by service, each endpoint pre-filled with **valid sample data** (seed users, real keys, correct headers) and a **"Send"** button that executes the call against localhost and pretty-prints the live response. Includes a login panel (token stored in memory, auto-attached), an auto-UUID idempotency-key button, and a guided "full journey" section that chains send → status → statement.

## Why / what you'll learn
Building the DX artifact that makes the whole project demoable in one click — and the practical web plumbing behind it: **CORS** (services must allow `null`/localhost origins for the explorer's fetch calls — add a dev-profile CORS config in common-lib, documented as dev-only), fetch with headers/JSON, rendering problem+json nicely, and designing *guided* API docs (each endpoint shows: description, the exact request about to be sent, editable body, response pane with status coloring, and links to the OpenAPI section). Why hand-rolled instead of Swagger UI: one file, zero deps, multi-service in one page (Swagger UI renders one spec/one server), and full control over the pre-filled happy-path data — plus writing it teaches the browser side of your own API.

## Prerequisites

## Tasks
1. Dev-only CORS configuration in common-lib (profile-gated, permissive for local origins; production posture note: never ship it).
2. `index.html` structure: sticky sidebar (services → endpoints), main pane cards per endpoint; embedded JSON catalog of endpoints (method, url template, headers, sample body, docs link) — generated by hand from openapi.yaml, kept in sync (checklist item in step's DoD and CLAUDE.md drift rule applies).
3. Behaviors: login card (alice/bob one-click) storing token in a JS var; UUID generator for Idempotency-Key with "reuse last key" toggle (demonstrates replay/409 live); Send buttons via fetch; response pane with status color, latency, formatted JSON; error rendering for problem+json.
4. Guided journey section: numbered cards that carry state (transactionId) forward; a chaos card wrapping mock-bacen config to demo the failure path from the browser.
5. Zero external network deps (no CDN — must work offline); keep it under ~1500 lines; `tools/api-explorer/README.md`.
6. Project closing pass: README sections current, PLAN.md fully checked, CHANGELOG release cut `0.1.0`.
